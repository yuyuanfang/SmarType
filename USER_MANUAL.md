## 1. 產品簡介

**快打 SmarType** 是一款全域語音輸入工具。無論你正在使用瀏覽器、Word、微信、VS Code 或任何其他應用程式，只要按住熱鍵說話，放開後文字就會自動輸入到游標所在位置。

### 主要特色

- **全域熱鍵**：在任何應用程式中按住 Right Shift 即可錄音，放開即辨識
- **即時跑馬燈**：說話過程中，辨識結果即時顯示在螢幕底部綠色膠囊中
- **智能語言切換**：根據目前使用的應用程式，自動切換繁體中文 / 簡體中文 / 英文 / 日文
- **中英翻譯**：在程式編輯器（VS Code、Cursor、Claude Code 等）中說中文，自動翻譯成英文輸出
- **智能詞典**：自動從語音歷史中學習高頻詞彙，提升辨識準確度
- **AI 潤色**：可選啟用 GPT-4o-mini 或 Gemini 2.5 Flash 去除贅字、修正同音字
- **語音日記**：每次語音輸入自動分類（工作 / 生活 / 財務 / 雜記），每晚 22:00 生成當日摘要
- **簡繁轉換**：內建詞彙級簡繁轉換（不只字符替換，支援兩岸用詞差異）

## 3. 安裝與啟動

### 首次安裝

1. 確認已安裝 Python 3.10+
2. 安裝相依套件：
   ```
   pip install pyaudio keyboard pystray pillow openai groq psutil pywin32
   ```
3. 編輯 `userdata/config.json`，填入 Groq API Key：
   ```json
   {
     "groq_api_key": "你的 Groq API Key"
   }
   ```

### 啟動方式

雙擊 `start.bat` 即可啟動。程式會：
1. 以管理員權限啟動（若未取得則自動提權）
2. 在系統匣（System Tray）顯示圖示
3. 自動開啟管理介面（Dashboard）

### 系統匣選單

在系統匣圖示上右鍵，可看到：

| 選項 | 說明 |
|------|------|
| 快打 SmarType · Groq OK | 顯示 API 連線狀態 |
| 顯示管理介面 | 開啟 Dashboard |
| 退出 | 完全關閉程式 |

## 4. 快速開始

1. 雙擊 `start.bat` 啟動
2. 確認系統匣出現綠色圖示
3. 在任意應用程式中：
   - **按住** Right Shift → 開始錄音（螢幕底部出現綠色膠囊）
   - **說話**（膠囊中即時顯示辨識文字）

> **提示：** 文字透過剪貼簿注入（Ctrl+V），所以游標必須在可輸入的位置。

## 5. 核心功能

### 5.1 語音辨識流程

```
按住熱鍵 → 錄音開始
  ├── 每 3 秒分段 → 本地 Whisper Tiny 即時辨識（跑馬燈）
  └── 放開熱鍵 → 完整音訊送 Groq Whisper Large v3 Turbo
       ├── 成功 → 文字輸出
       └── 失敗 → 備用 OpenAI Whisper-1
            └── 再失敗 → 顯示錯誤
```

### 5.2 即時跑馬燈

錄音過程中，每 3 秒將已累積的音訊片段送到本地 Whisper Tiny 模型進行快速辨識（約 150ms），辨識結果即時顯示在螢幕底部的綠色膠囊中。這讓你在說話時就能看到初步辨識結果。

> **注意：** 跑馬燈使用的是輕量級本地模型，準確度較低。最終貼上的文字是由 Groq 雲端大模型辨識的結果，準確度更高。

### 5.3 幻覺過濾

系統內建幻覺偵測機制，會自動過濾以下情況：
- 重複字詞（如「誠誠誠誠」）
- 已知的幻覺模式（如「字幕志願者」「Subtitles by」等）
- 靜音產生的假辨識

### 5.4 靜音偵測

錄音完成後，系統會檢查音訊的 RMS 音量。若低於設定的閾值，會直接跳過辨識，顯示「未偵測到語音」。這能有效防止按錯熱鍵時產生的幻覺文字。

### 5.5 簡繁轉換

系統根據目標語言自動進行簡繁轉換：
- **zh-TW**：簡體 → 繁體（含詞彙級轉換，如「軟件」→「軟體」）
- **zh-CN**：繁體 → 簡體
- **en**：不轉換（觸發翻譯功能）

#### 麥克風測試

點擊「🎤 測試麥克風」按鈕開始即時監測麥克風音量。測試會在 **15 秒後自動停止**，避免無謂的資源消耗。

#### 自動校準

點擊「⚡ 自動校準」，系統會錄製 2 秒背景噪音，自動計算建議閾值（= 環境噪音 RMS × 2）。

| 欄位 | 說明 |

系統會自動從每次語音輸入中學習高頻詞彙，以表格方式顯示（按頻次排序）。

## 8. 中英翻譯功能

### 觸發條件

當目標語言為 **`en`** 且辨識結果包含中文字時，系統會自動將中文翻譯成英文再輸出。

### 使用場景

在 VS Code、Cursor、Claude Code、終端等開發工具中說中文，系統會：
1. 用 Groq Whisper 辨識你的中文語音
2. 調用 GPT-4o-mini（或 Gemini 2.5 Flash）翻譯成英文
3. 將英文結果貼上到游標位置

### 翻譯模型優先順序

```
1. OpenAI GPT-4o-mini（若有 OpenAI API Key）
2. Gemini 2.5 Flash（若有 Gemini API Key）
3. 翻譯失敗 → 輸出原始中文
```

### 自訂翻譯規則

若想讓某個應用程式觸發翻譯，在 Dashboard 的「設定 → 視窗語言規則」中新增規則，語言選擇 `en`。

## 9. 智能字典系統

### 自動學習

每次語音辨識完成後，系統會在背景自動提取辨識文字中的詞彙：
- 中文詞彙：2~6 個字
- 英文詞彙：3 個字母以上
- 出現 2 次以上的詞彙會被加入智能字典

### 提升辨識率

字典中的高頻詞彙會被自動加入 Whisper 的提示詞（prompt），幫助辨識模型更準確地辨識這些詞彙。系統最多取前 10 個高頻詞，並將提示詞控制在 120 字元以內。

### 手動管理

在 Dashboard 的「字典」頁面：
- **新增詞彙**：輸入框輸入後點「+ 新增」
- **修改詞彙**：選中後點「修改」
- **刪除詞彙**：選中後點「刪除」
- **糾正映射**：設定「聽到 → 改成」的自動替換規則

### 資料儲存

| 檔案 | 內容 |
|------|------|
| `userdata/smart_dict.json` | 詞彙頻率（`{"words": {"詞彙": 次數}}`)  |
| `userdata/vocabulary.json` | 手動詞彙 + 糾正映射 + 統計 |

## 10. 語音日記

### 運作方式

1. 每次語音辨識完成後，文字會自動寫入當日日記
2. 若啟用 AI 潤色，會先去贅字、修正同音字，再分類
3. AI 自動分類：工作 / 生活 / 財務 / 雜記
4. 每晚 22:00 自動生成當日摘要

### 日記檔案

日記儲存在 `userdata/diary/` 目錄，格式：
```
userdata/diary/2026-03-28.jsonl
```

每行一筆紀錄：
```json
{"time": "14:30:25", "category": "work", "text": "明天要交報告", "window": "chrome.exe"}
```

### 在 Dashboard 中查看

1. 進入「語音日記」頁面
2. 選擇日期（預設今天）
3. 點「載入」
4. 用分類標籤篩選（全部 / 工作 / 生活 / 財務 / 雜記 / 摘要）

## 13. 疑難排解

### 常見問題

#### Q: 系統匣圖示不出現

**可能原因：**
- `start.bat` 語法錯誤
- `userdata/vocabulary.json` 檔案損壞
- Python 未正確安裝

**解決方式：**
1. 在命令列中手動執行 `python dictation.py` 查看錯誤訊息
2. 檢查 `userdata/debug.log` 中的錯誤紀錄
3. 若 `vocabulary.json` 損壞，刪除後重啟（系統會自動重建）

#### Q: 辨識結果為空

**可能原因：**
- 麥克風閾值設太高（RMS 低於閾值被當作靜音）
- Groq API Key 無效或額度已滿
- 網路連線問題

**解決方式：**
1. 在 Dashboard 首頁點「🎤 測試麥克風」確認 RMS 數值
2. 調低閾值或點「⚡ 自動校準」
3. 檢查 API Key 是否正確

#### Q: 辨識出奇怪的文字（幻覺）

**可能原因：**
- 環境噪音太大
- 錄音太短（< 0.5 秒）

**解決方式：**
1. 提高閾值以過濾背景噪音
2. 說話時間拉長，確保超過 1 秒

#### Q: Dashboard 閃退

**可能原因：**
- 有多個 SmarType 實例同時運行

**解決方式：**
1. 在任務管理器中結束所有 `python.exe` 進程
2. 重新雙擊 `start.bat`

#### Q: 翻譯功能未觸發

**可能原因：**
- 目標應用程式不在英文規則清單中
- `app_rules.json` 中的自訂規則覆蓋了內建規則

**解決方式：**
1. 在 Dashboard「設定 → 視窗語言規則」中為該應用程式新增 `en` 規則
2. 或手動編輯 `userdata/app_rules.json`

### 日誌檔案

| 檔案 | 說明 |
|------|------|
| `userdata/debug.log` | 完整執行日誌（辨識結果、錯誤、API 呼叫） |
| `userdata/history.jsonl` | 所有辨識歷史紀錄 |
| `userdata/status.json` | 目前系統狀態（idle / recording / processing） |

### 診斷技巧

1. 開啟 `userdata/debug.log`，搜尋 `ERROR` 或 `EXCEPTION`
2. 若每個事件都出現兩次（時間差 < 5ms），代表有多個實例同時運行 → 結束所有進程後重啟
3. 若 Groq 報錯 `prompt length must be 896 characters or fewer`，代表提示詞過長 → 系統已內建 120 字元硬截斷保護
